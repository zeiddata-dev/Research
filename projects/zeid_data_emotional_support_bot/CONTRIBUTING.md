# CONTRIBUTING

## Development

1. Edit training data in `data/emotion_training.jsonl`
2. Improve code in `emotion_bot/`
3. Retrain and evaluate
4. Keep comments readable and kind
5. Do not turn the bot into a manipulative pickup artist with a JSON file

## Suggested style

- Keep functions small
- Prefer clear names over clever names
- Add tests for new behavior
- Preserve the narrative comments unless you are replacing them with better ones
